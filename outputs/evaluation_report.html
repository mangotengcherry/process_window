L1 → L3 yield 예측 + SPEC window 추천 프로토타입의 모델/추천 품질 평가. 모델 정확도 (1부) 와 추천 품질 (2부) 을 함께 본다.

⚠ 본 추천은 통계적 상관관계 기반이며 인과관계가 아니다. 실제 SPEC 변경 전 도메인 엔지니어 검토 + pilot lot 검증 필수.

Predicted vs Actual

점이 빨간 대각선에 모일수록 예측이 정확. 큰 산포 = 노이즈/누락 feature.

Residual Plot

잔차가 0 주변에서 무작위면 unbiased. 한쪽으로 치우치거나 특정 예측구간에서 변동성이 커지면 모델 보완 필요.

Feature Importance

Permutation importance — 해당 feature 셔플 시 yield 예측 성능이 얼마나 떨어지는지. 추천 결과 해석의 1차 근거.

2. 추천 품질

단순한 R² 보다 추천이 yield 개선에 기여하는가 와 현장에 적용 가능한가 (coverage 손실 / segment 편향) 를 동시에 본다.

Recommendation Impact (uplift × coverage loss)

좌상단 (낮은 coverage loss + 높은 uplift) 일수록 가치 있는 추천. 색상은 Confidence (A=초록 최선 / B=주황 / C=회색 reference).

Score Breakdown

각 추천의 final score 가 어떤 구성요소 (uplift + bonus − penalties) 로 나왔는지. 검은 다이아몬드 = 최종 점수.

Current vs Recommended Window

feature 별 현재 (회색) vs 추천 (초록) [LSL .. USL] 범위. × = target. 이동 방향, 좁힘/넓힘이 직관적으로 보인다.
